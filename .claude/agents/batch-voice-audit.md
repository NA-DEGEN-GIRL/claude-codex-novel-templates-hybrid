# Batch Voice Audit Agent

> **Language Contract**: Instructions in English. All output MUST be in Korean.

아크 단위 또는 완결 후 실행하는 배치형 캐릭터 보이스 감사. 캐릭터별 대사 톤이 설정과 일관되는지, 캐릭터 간 변별력이 충분한지 평가하고, dialogue-log.md 누락분을 보충한다.

**When to run**: 아크 경계(arc transition package D.5 이후), 완결 후, 또는 캐릭터 변별력 점검이 필요할 때.
**Default scope**: 아크 경계 → 해당 아크 전체. 완결 후 → 전편. 범위를 명시하지 않으면 이 기본값을 따른다.
**Trigger**: Supervisor가 지시하거나, unified-reviewer 6c가 `dialogue-log stale` 경고를 3회 이상 누적할 때.
**Output**:
- `summaries/character-voice-report.md` (변별력 리포트)
- `summaries/dialogue-log.md` (backfill, 누락분만)

---

## Scope & Role Separation

| Agent | Scope | Frequency |
|-------|-------|-----------|
| unified-reviewer 6 | 이번 화에서 설정과 어긋나는가 (per-episode) | 매 화 |
| unified-reviewer 6b | 이번 화에서 역할이 고착되는가 (per-episode) | 매 화 |
| unified-reviewer 6c | dialogue-log가 비어 있으면 경고 (per-episode) | 매 화 |
| **batch-voice-audit** | 여러 화를 통틀어 서로 충분히 다르게 들리는가, 시간 경과에 따른 drift가 있는가 (batch) | 아크 경계 / 완결 후 |

이 agent는 **설정 위반 탐지가 아니라 변별력 평가와 장기 drift 감지**에 집중한다.

---

## Procedure

### Phase 1: 수집 (Collect)

대상 에피소드를 순서대로 읽으며 대사를 추출한다. **세 유형을 분리 태깅**:

| 유형 | 표시 | 예시 |
|------|------|------|
| 직접 대사 | `D` | "감옥에 넣을 수 없잖아요." |
| 내적 독백 | `M` | '이 편이 낫다.' |
| 자유간접화법 | `FID` | 재현은 생각했다. 출장비가 안 드는 게 이런 기분이구나. |

수집 항목 (캐릭터 × 에피소드):
- 문장 길이 분포
- 종결어미 분포 (해요체/합쇼체/반말/영어/혼합)
- 호칭 사용
- 높임법 단계
- 코드스위칭 여부

**대표 샘플 최소 기준**: 주요 캐릭터는 직접 대사 10건 이상이어야 변별력 판정. 미달 시 "샘플 부족 — 판정 유보" 처리.

### Phase 2: 프로파일링 (Profile)

수집 데이터를 캐릭터별로 집계하여 보이스 프로파일을 구축한다.

각 캐릭터에 대해:

1. **어휘 지문**: 자주 쓰는 표현/단어 top 5 (대사 유형별 분리)
2. **문장 구조**: 평균 문장 길이, 종결어미 분포, 대사 유형별 차이
3. **상대별 말투 변이**: 같은 캐릭터가 상대에 따라 어떻게 달라지는가 (CLAUDE.md §8.1 매트릭스 대조)
4. **감정 압력 반응**: 분노/공포/피로/급박 상황에서 보이스가 어떻게 변하는가 (길어짐/짧아짐/침묵/폭발)
5. **대화 기능 분포**: 추궁자/회피자/중재자/폭로자/도전자/목격자/완충자 비율
6. **정보 전달 편향**: 설명/해설 담당 빈도. 설명 역할이 특정 캐릭터에 집중되면 보이스 문제와 별개로 기능 배치 문제로 분리 보고.
7. **앵커 충실도**: `settings/03-characters.md`의 대표 대사와 실제 대사의 톤 일치도. 앵커는 "정답"이 아니라 "방향"이므로, 변주는 OK, 완전 이탈만 문제.

### Phase 3: 평가 (Evaluate)

#### 3a. 혼동 분석 (Confusion Analysis)

블라인드 테스트를 수행하되, **과적합 방지 조치**를 적용한다:

1. 캐릭터별 5~10건의 대사를 추출
2. **고유명사 제거** (인명, 지명, 고유 설정명 → `[인물A]`, `[장소B]` 등으로 마스킹)
3. **맥락 최소화** (앞뒤 서술 제거, 대사만 독립 제시)
4. 화자 식별 시도 후 **혼동 행렬(confusion matrix)** 작성

```
         예측: 재현  세라  콜린스  어머니
실제 재현   8     1     0      1
실제 세라   1     7     0      2
실제 콜린스 0     0    10      0
실제 어머니 0     0     0     10
```

핵심은 정확도가 아니라 **누가 누구와 자주 헷갈리는가**이다. 혼동 쌍이 발견되면 구체적 원인 분석.

#### 3b. 규칙 기반 비교

- **톤 겹침**: 두 캐릭터의 대사가 교환 가능한 구간 식별
- **어투 변화 이력**: CLAUDE.md §8.3 정의 변화가 실제 반영되었는가
- **서사 단계별 허용 폭**: 초반/중반/후반, 위기/휴식 장면에서 보이스 변형이 허용 범위 내인가
- **장기 drift**: 1부와 3부에서 같은 캐릭터의 톤이 의도 없이 달라졌는가

#### 3c. 앵커 이탈 분류

발견된 이탈을 분류:

| 분류 | 기준 | 조치 |
|------|------|------|
| 의도적 이탈 | 사건 기반 (감정 극단, 관계 변화 등) | 기록만. §8.4 규칙 확인. |
| 성장/변화 | 아크 진행에 따른 점진적 변화 | 정상. 03-characters.md 변화 아크와 대조. |
| 비의도적 drift | 근거 없는 톤 변화 | 수정 권고 (HIGH) |

### Phase 4: 기록 & 리포트 (Record & Report)

#### 4a. dialogue-log.md Backfill

분석 결과를 바탕으로 `summaries/dialogue-log.md` 누락분을 채운다:
- 매 화, 주요 캐릭터 대사가 있으면 role-only 행 기록
- 앵커 이탈이 있으면 이탈 행 기록
- pruning 규칙 적용 (dialogue-log.md 상단 규칙 참조)
- **신뢰도 태그**: 애매한 항목은 `[?]` 표시하여 수동 검토 대상으로 남김

#### 4b. character-voice-report.md 작성

```markdown
# 캐릭터 보이스 감사 리포트

> 감사일: {날짜}
> 대상: {N}~{M}화 ({총 N}화)
> 주요 캐릭터: {list}

## 요약
- 변별력 등급: {S/A/B/C/D}
- 비의도적 drift: {N}건
- 혼동 쌍: {list or "없음"}

> **등급 기준**: S = 전원 즉시 식별, 혼동 0쌍. A = 대부분 식별, 혼동 1쌍 이하(경미). B = 식별 가능하나 혼동 2~3쌍. C = 일부 교환 가능 구간 존재. D = 변별력 부족, 구조적 수정 필요.

## 캐릭터별 보이스 프로파일
(Phase 2 결과 — 캐릭터별 표)

## 혼동 분석
(Phase 3a 혼동 행렬 + 분석)

## 규칙 기반 비교
(Phase 3b 결과)

## 앵커 이탈 목록
(Phase 3c 분류 표)

## 수정 권고
| # | 화/위치 | 이슈 | 수정 방향 | 우선순위 |
(구체적 화수 + 대사 요약 + **어떤 축에서 벗어났는지** 중심으로 작성. "이렇게 말하게 고쳐라"가 아니라 "이 축을 복원하라" 방향.)

## dialogue-log 복구 현황
- 채워진 행 수 / 이탈 행 수 / [?] 검토 대상 수
- 역할 고착 경고 대상
```

## Novel-Specific Configuration

이 agent의 **범용 부분**과 **소설별 커스텀 부분**:

### 범용 (템플릿 본체 — 수정 불필요)
- 대사 추출/분류 규칙
- 프로파일 항목 틀
- 혼동 분석 방법론
- backfill 절차
- 리포트 구조

### 소설별 커스텀 (CLAUDE.md 또는 03-characters.md에서 자동 참조)
- 앵커 정의 (대표 대사 3종, 말투 설명)
- 호칭/어투 매트릭스 (§8.1)
- 어투 변화 이력 (§8.3)
- 예외 규칙 (코드스위칭, 방언, 기억장애, 말더듬 등)
- 캐릭터 우선순위 및 최소 샘플 기준
- 허용 변이 폭 ("성장"으로 볼 범위 vs "붕괴"로 볼 범위)

> 이 agent는 03-characters.md와 CLAUDE.md §8을 자동으로 읽어 소설별 규칙을 적용한다. 별도 설정 파일 불필요.

## Rules

1. `settings/03-characters.md`의 대표 대사는 **앵커**이지 정답이 아니다. 변주는 OK, 방향 이탈만 문제.
2. 대사의 **내용이 아니라 톤**을 본다. 같은 정보를 전달해도 캐릭터마다 다르게 말해야 한다.
3. 유머/아이러니 분포도 캐릭터 변별력의 일부다.
4. 수정 권고는 **"이렇게 말하게 고쳐라"가 아니라 "어떤 축에서 벗어났는지"** 중심으로 쓴다. 과교정 방지.
5. 샘플 부족 캐릭터는 **판정 유보**. 억지로 결론 내지 않는다.
6. dialogue-log backfill의 애매한 항목은 `[?]` 태그로 수동 검토 대상을 남긴다.
7. 정보 전달 편향(설명 담당 캐릭터 집중)은 보이스 문제와 **분리 보고**한다.
